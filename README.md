# RobinHood P2P v5.4.2

Децентрализованный P2P-мессенджер с оконечным шифрованием, построенный на платформе **P2PPong**. Работает в браузере. Без установки, без центрального сервера хранения сообщений.

## Архитектура

Приложение работает поверх платформы **P2PPong** — распределённого P2P-ядра, построенного на принципах ОГАС академика Глушкова.

### Три уровня сигнальной сети (ОГАС)

| Уровень              | Технология                      | Назначение                               |
|----------------------|---------------------------------|------------------------------------------|
| Главный центр        | Cloudflare Worker (WebSocket)   | Быстрая адресная маршрутизация маяков    |
| Опорные центры       | Render HTTP Server (polling)    | Резервный канал при блокировке WebSocket |
| Терминалы            | Пиры в браузерах (DHT + WebRTC) | Прямые соединения без серверов           |

**Слепой Улей:** Серверы не читают содержимое сообщений, не хранят историю, не знают кто с кем общается. Только пересылают зашифрованные блобы конкретному адресату. **Никакого broadcast.**

## Идентификация

**Peer ID** = первые 32 символа SHA-256 от конкатенации:

- Параметров WebGL-рендерера (вендор + модель GPU)
- Характеристик AudioContext (частота дискретизации, число каналов)
- Размеров экрана и глубины цвета
- `navigator.hardwareConcurrency` и `deviceMemory`
- Случайной соли (генерируется однократно, хранится в localStorage)

Стабильный, но не привязанный к личности идентификатор устройства.

## Сквозное шифрование (E2EE)

### Рукопожатие
ECDH на кривой P-256. Каждая сторона генерирует ephemeral-пару, обменивается публичными ключами через маяки (blind beacon). **Маяки доставляются только целевому пиру**, а не всем подряд.

### Double Ratchet + HMAC
- **Ratchet Key** обновляется каждое сообщение: `SHA-256(старый_ключ + индекс)`
- **HMAC-SHA256** подписывает каждое сообщение перед шифрованием
- **Старые ключи** сохраняются (до **50**) для расшифровки сообщений, пришедших не по порядку
- **Ресинхронизация:** при расхождении ratchet — автоматический перезапуск через ECDH
- **Защита от replay-атак:** проверка `ratchetIndex`

### Шифрование сообщений
- **AES-256-GCM**, случайный IV (12 байт)
- **GZIP-сжатие** перед шифрованием
- **Случайный паддинг** (20–70 байт) внутри сообщения
- **Фиксированный размер блоба:** 4096 байт (padding против анализа размера)

### Индикатор Ratchet
В статус-баре отображается количество оборотов ratchet:
- 🔄 0–9 — инициализация
- 🔒 10–49 — активный
- 🔐 50+ — безопасный

## Маяки и каналы

| Сущность  | Время жизни                         | Описание                                 |
|-----------|-------------------------------------|------------------------------------------|
| Маяк      | 5 минут                             | Зашифрованное объявление с targetPeerId  |
| Канал     | 10 минут (продлевается активностью) | Зашифрованный туннель между двумя пирами |

### Процесс подключения

1. **Устройство А:** открывает Колчан (кнопка 🏹), вводит Peer ID друга, жмёт «Создать маяк»
2. **Устройство Б:** получает маяк, автоматически создаёт канал, отправляет beacon-response
3. На устройстве Б загорается кнопка «Заявки» — пользователь нажимает для подтверждения
4. На устройстве А приходит beacon-response — канал создан
5. **Оба видят:** `✅ Канал открыт!`, контакт появляется в Лучниках

## Сообщения и звонки

### Текстовые сообщения
Шифруются блобом, отправляются через DHT (предпочтительно) или WS (fallback). При получении автоматически отображаются в чате.

### Голосовые сообщения
- Запись через MediaRecorder (Opus в WebM, 16 kbps)
- Отображаются с аудиоплеером (play/pause, волновая анимация, таймер)
- Шифрование и отправка блобом с тегом `voice: true`

### Голосовые вызовы
- Полный WebRTC-стек с рингтоном и КПВ
- Кнопки «Ответить»/«Отклонить» при входящем вызове
- ICE-кандидаты буферизируются и отправляются группой

## Фоновая P2P-помощь (PeerHelp)

Каждый пир становится микро-сидом сети:
- Анонсирует себя на WebTorrent-трекерах
- Потребляет ~2-5 КБ/час
- Работает даже когда вкладка скрыта
- Помогает новым пирам найти сеть
