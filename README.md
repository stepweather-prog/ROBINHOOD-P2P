# RobinHood P2P v5.5.4

Децентрализованный P2P-мессенджер с оконечным шифрованием (одноразовые сеансы, без сохранения ключей между сессиями). Работает в браузере. Без установки, без центрального сервера хранения сообщений.

## Модель угроз

**Проект защищает от:**
- Чтения сообщений третьей стороной (AES-256-GCM + ECDH P-256 + Double Ratchet)
- Подмены сообщений (HMAC-SHA256 на каждом пакете)
- Повторного воспроизведения старых сообщений (одноразовые nonce + ratchet)
- Раскрытия истории при компрометации ключа (forward secrecy — sending-ключи не сохраняются)
- Связывания сессий по Peer ID (beaconId скрывает идентификатор от сервера)
- Массового сбора метаданных в публичном пуле (сервер не знает, какой маяк кому принадлежит)

**Проект НЕ защищает от:**
- Корреляции пиров по IP-адресам и времени запросов (сервер видит, что два IP обратились к одному ключу в близкое время; в режиме публичного пула корреляция ослаблена)
- MitM при компрометации сигнального сервера (код верификации передаётся через сервер, не вне полосы; рекомендуется подтверждать код голосом или при встрече)
- Анализа паттернов трафика (частота и размер пакетов видны сетевому наблюдателю)
- Атак на конечные устройства (вредоносное ПО, кейлоггеры, захват браузера)
- Рассинхрона ratchet при одновременной отправке сообщений обеими сторонами

## Архитектура

Приложение работает поверх платформы **P2PPong** — распределённого P2P-ядра, построенного на принципах ОГАС академика Глушкова.

RobinHood UI (интерфейс) — Анимации, темы, чат, звонки. НЕ знает про ECDH, Ratchet, DHT.

P2PPong (ядро) — Криптография, транспорт, каналы. НЕ знает про document, DOM, UI. Событийная шина: P2PPong.on() / P2PPong.emit().

Транспорт — Cloudflare Worker (основной), Render Server (резервный), WebRTC DataChannel (прямой канал). Публичный пул маяков: `/pool` на обоих серверах.

## Режимы соединения

**Обычный маяк** — beaconId как ключ на сервере. Peer ID скрыт от сервера.

**Публичный колчан** — маяк в общем пуле `/pool`. Джойнер скачивает все маяки и расшифровывает. Сервер не знает, какой маяк кому принадлежит.

**Тайный колчан** — beaconId = SHA-256(секрет + соль). Только создатель и джойнер могут вычислить ключ. Требует обмена секретом вне полосы.

## Схема рукопожатия

Пир А (создатель): craftArrow() → генерирует PeerID, ECDH, beaconId → SHA(pubKey+'beacon') → ключ маяка → AES-256-GCM(inner, ключ) → шифрует peerId, beaconId, код, ник, аватар → POST /beacon waiting_<beaconId> или POST /pool.

Пир Б (присоединяющийся): joinBeacon(beaconId) → GET /beacon waiting_ → получает маяк → проверяет HMAC(pubKey+peerId) → SHA(pubKey+'beacon') → ключ → расшифровывает inner. Или joinPublicPool() → GET /pool → перебирает все маяки → расшифровывает каждый → находит подходящий.

Верификация: 7-значный код (crypto.getRandomValues). Пир Б отправляет код через code_<beaconId>. Пир А сверяет. При совпадении канал открывается автоматически. Подтверждение через QR-код или голос (Web Speech API). Предупреждение в UI: «Сравни код с собеседником голосом или при встрече.»

Канал открыт. Раздельные sending/receiving ratchet + AES-256-GCM. WebRTC DataChannel (если доступен).

## Идентификация

Peer ID = 32 случайных hex-символа (криптографически безопасный PRNG). Генерируется заново при каждом craftArrow(). Не привязан к устройству, не сохраняется между сессиями.

Beacon ID = случайный идентификатор маяка. Используется как ключ на сигнальном сервере. Peer ID скрыт от сервера. Передаётся между пользователями для соединения.

## Сквозное шифрование (E2EE)

Рукопожатие: ECDH на кривой P-256. Ephemeral-пары. Обмен публичными ключами через маяки.

Маяки: Ключ маяка = SHA-256(pubKey + 'beacon'). HMAC-SHA256 подпись = HMAC(pubKey + peerId, ключ маяка). Получатель проверяет HMAC → расшифровывает AES-256-GCM → извлекает код и данные.

Раздельные Ratchet: sendKey/sendIndex для исходящих. recvKey/recvIndex для входящих. Sending-ключ обновляется при каждой отправке. Receiving-ключ прокручивается до нужного индекса при получении. Старые receiving-ключи сохраняются (до 3) для восстановления порядка. Sending-ключи не сохраняются (forward secrecy). HMAC-SHA256 подписывает каждое сообщение. Автоматическая ресинхронизация при расхождении.

Шифрование сообщений: AES-256-GCM, случайный IV (12 байт). GZIP-сжатие перед шифрованием. Случайный паддинг (20–70 байт).

## Верификация

7-значный цифровой код (crypto.getRandomValues). Подтверждение: ввод кода + опционально QR / голос. Предупреждение в UI: «Сравни код с собеседником голосом или при встрече. Не доверяй только экрану.»

## Голосовые сообщения

Максимальный размер: 50 КБ. Максимальная длительность: 10 секунд. Кодек: Opus в WebM.

## Формат данных

Маяк (beacon): тип, pubKey, peerId, inner (AES-256-GCM), sig (HMAC-SHA256). Nonce удалён из открытой части.

Сообщение (блоб): z (GZIP), t (время), n (nonce), ri (ratchet index).

## API для разработчиков

P2PPong.init() — инициализация.
P2PPong.craftArrow() — создать обычный маяк, возвращает beaconId.
P2PPong.craftPublicArrow() — создать маяк в общем пуле.
P2PPong.craftSecretArrow(secret) — создать маяк с хэшем от секрета.
P2PPong.joinBeacon(targetBeaconId) — присоединиться к маяку.
P2PPong.joinPublicPool() — найти маяк в общем пуле.
P2PPong.joinSecretBeacon(secret, salt) — присоединиться по секрету.
P2PPong.confirmVerification() — подтвердить верификацию.
P2PPong.sendMessage(channelId, text) — отправить сообщение.
P2PPong.sendVoiceMessage(channelId, audioBase64) — отправить голосовое.
P2PPong.getBeaconId() — получить beaconId.
P2PPong.getVerificationCode() — получить код верификации.
P2PPong.on('ready', () => {}) — ядро готово.
P2PPong.on('peer-id-generated', ({ peerId, beaconId, code }) => {}) — маяк создан.
P2PPong.on('verification-needed', ({ code }) => {}) — нужно ввести код.
P2PPong.on('verification-received', ({ code }) => {}) — код получен.
P2PPong.on('channel-opened', ({ channelId, peerId, nick, avatar }) => {}) — канал открыт.
P2PPong.on('message-received', ({ channelId, text, timestamp, nick, avatar }) => {}) — сообщение получено.
P2PPong.on('message-sent', ({ channelId, data, status }) => {}) — сообщение отправлено.
P2PPong.on('beacon-timeout', () => {}) — таймаут ожидания маяка.
P2PPong.on('error', ({ message }) => {}) — ошибка.

## Требования к браузеру

HTTPS (обязательно). Web Crypto API (ECDH, AES-GCM, HMAC-SHA256). WebRTC (опционально). CompressionStream API. Service Worker (опционально).

## Известные ограничения

Нет офлайн-доставки сообщений (маяк живёт 5 минут). Нет групповых чатов (только P2P). Нет push-уведомлений когда вкладка закрыта. Нет роуминга истории между устройствами. WebRTC может не работать за симметричным NAT без TURN-сервера. Сервер видит корреляцию по IP и времени (в режиме публичного пула корреляция ослаблена). Код верификации передаётся через сигнальный сервер (не вне полосы). При одновременной отправке сообщений возможен рассинхрон ratchet. Криптографические решения не прошли независимый аудит.
